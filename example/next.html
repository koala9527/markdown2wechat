手把手教你安装和测试n8n：强大的开源自动化工具

什么是n8n？

n8n（发音为"n-eight-n"）是一个功能强大的开源工作流自动化工具，可以连接各种应用程序、API和服务。它类似于Zapier或Make，但完全免费且可以自托管，让你完全掌控自己的数据。

为什么选择n8n？

✅ 完全开源 - 基于公平代码许可 ✅ 自托管能力 - 数据完全由你掌控 ✅ 可视化工作流 - 拖拽式界面，无需编码 ✅ 丰富的集成 - 支持300+应用程序和服务 ✅ 高度可定制 - 支持自定义节点和功能

安装方法

方法一：使用Docker（推荐）

# 创建数据目录
mkdir ~/.n8n

# 运行n8n容器
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  n8nio/n8n

方法二：使用npm

# 安装n8n
npm install n8n -g

# 启动n8n
n8n start

方法三：使用Docker Compose

创建docker-compose.yml文件：

version: '3.8'

services:
  n8n:
    image: n8nio/n8n
    container_name: n8n
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=password
    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:

运行：

docker-compose up -d

访问和初始设置

  1. 打开浏览器访问：http://localhost:5678
  2. 首次访问需要创建管理员账户
  3. 设置用户名、密码和邮箱
  4. 完成初始配置

创建你的第一个工作流

示例：GitHub Issues自动通知到Slack

让我们创建一个简单的工作流，当GitHub仓库有新issue时，自动发送通知到Slack。

步骤1：添加GitHub触发器

  1. 点击"添加第一个工作流"
  2. 搜索并选择"GitHub"节点
  3. 选择"On Issue"触发器
  4. 配置GitHub连接（需要GitHub Token）

步骤2：添加Slack节点

  1. 从GitHub节点拖出连接线
  2. 搜索并选择"Slack"节点
  3. 选择"Send Message"操作
  4. 配置Slack连接（需要Slack Token）

步骤3：配置消息内容

// 在Slack节点的消息字段中
`📢 新Issue创建！
仓库：{{$node["GitHub"].json.repository.full_name}}
标题：{{$node["GitHub"].json.issue.title}}
链接：{{$node["GitHub"].json.issue.html_url}}`

步骤4：激活工作流

  1. 点击右上角的"激活"开关
  2. 工作流现在开始监听GitHub事件

常见问题解决

问题1：端口冲突

# 更改端口
n8n start --port=5679
# 或
docker run -p 5679:5678 n8nio/n8n

问题2：连接数据库

# 使用外部数据库
export DB_TYPE=postgresdb
export DB_POSTGRESDB_HOST=localhost
export DB_POSTGRESDB_PORT=5432
export DB_POSTGRESDB_DATABASE=n8n
export DB_POSTGRESDB_USER=n8n
export DB_POSTGRESDB_PASSWORD=your_password

问题3：内存不足

# 增加内存限制
docker run -it --rm \
  --memory="1g" \
  --memory-swap="2g" \
  n8nio/n8n

安全配置建议

  1. 启用认证
export N8N_BASIC_AUTH_ACTIVE=true
export N8N_BASIC_AUTH_USER=admin
export N8N_BASIC_AUTH_PASSWORD=secure_password
  1. 使用HTTPS
export N8N_PROTOCOL=https
export N8N_SSL_KEY=/path/to/key.pem
export N8N_SSL_CERT=/path/to/cert.pem
  1. 限制访问IP
export WEBHOOK_URL=https://your-domain.com
